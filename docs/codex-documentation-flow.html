How Codex Reads This Repository

The document path used when a new converter-project thread begins.

1 New thread starts

Codex establishes the repository root and reads the instructions before planning or running project commands.

2 Read every root Markdown file

  • AGENTS.md defines the working agreement and completion requirements.
  • CLAUDE.md redirects contributors to that agreement.
  • README.md gives the project overview and entry point.

3 Follow the required entry path

  • docs/START_HERE.md gives the reading order and current decisions.
  • docs/README.md maps task types to authoritative documents.
  • docs/project/ROADMAP.md identifies priorities and acceptance criteria.

4. What kind of task was requested?

Codex now takes the shortest relevant documentation path.

5 Inspect, implement, and verify

Codex reads the relevant code and history, performs the task, and runs tests proportional to the affected behavior.

6 Update documentation with the change

Current-state documentation is always updated when behavior or instructions change. Meaningful changes may also require a change note, engineering-log event, lab note, ADR, or roadmap update.

7 Complete repository checks

  • Run python tools\docs_index.py --write.
  • Run python tools\docs_index.py --check.
  • Re-read AGENTS.md completion requirements.
  • Report implementation files, documentation files, verification, and unresolved risks.
Important: Codex does not read the entire documentation tree on every task. It reads the mandatory entry documents first, then follows the task-specific route defined by docs/README.md.
