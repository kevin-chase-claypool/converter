RP23CNC Homing Data Flow Sheet

Goal: the normal startup experience should be simple. The user connects ioSender, confirms the pen is lifted, runs grblHAL homing, and the system establishes X, Y, and A references without a manual magnetic scan.

Required Components

Windows PC + ioSender

Starts homing and reports status. It does not interpret magnetic fields during normal startup homing.

USB or Telnet $H / $HA status

RP23CNC / grblHAL

Owns X/Y/A motion, homing cycles, step generation, limit inputs, and machine coordinates.

X/Y switches A_HOME G-code

RP2040 + TMAG5273

Reads the magnetic gradient and turns it into a clean digital A home/index signal.

I2C/Qwiic hysteresis A_HOME

Toolhead Controller

Receives M5/M3. Must keep the pen retracted during homing and calibration.

M5 = lift M3 = engage safe default

Normal Startup Homing Overview

Color references in this overview match the detailed sections below: Bed-Center Calibration Flow is a post-homing calibration path from the homing-complete state, while A-Axis Magnetic Homing Subflow is the sensor-driven A_HOME portion inside normal homing.

Bed-Center Calibration Flow

This flow locates the physical bed center after X/Y are homed. It is a separate magnetic operation from A/theta homing. The TMAG5273 scans a bounded 4 in x 4 in X/Y window around the expected center magnet, finds the left/right and front/back magnetic footprint edges, and computes the actual bed-center point.

Requirement Specific Need Owner
Starting state Pen/toolhead is lifted with M5; X/Y are homed using physical switches before magnetic center scanning starts. ioSender, grblHAL, toolhead controller
Scan window The TMAG5273 scans a bounded 4 in x 4 in X/Y window around the expected bed-center magnet. The footprint must not fill or leave the window. grblHAL motion command and calibration logic
Sensor readings RP2040 records TMAG5273 readings at each X/Y sample point, using averaged samples and the fixed installed sensor height. RP2040/TMAG5273 firmware
Center computation Use thresholded or saturated footprint edges, not the peak field value: center_x = (left_edge + right_edge) / 2 and center_y = (front_edge + back_edge) / 2. Setup calibration script or future adapter logic
Output The result is the measured bed center used to position the sensor at the outer magnet radius before the A/theta scan. The center magnet does not define A angular phase. Calibration workflow

A-Axis Magnetic Homing Subflow

This flow is for the A/theta axis when the system must determine both bounds of the outer magnet field. Because the converter and grblHAL convention uses motor-shaft degrees, one full bed revolution is 4320 A degrees with the current 12:1 bed ratio. Normal A homing should scan two full bed revolutions, or 8640 A motor degrees, record two valid entry/exit pairs, verify they agree, and average the two computed centers.

Requirement Specific Need Owner
Normal A-homing scan distance Scan two full bed revolutions. With the current 12:1 convention, this is exactly 8640 A motor degrees. grblHAL motion command and RP2040 edge recorder
Position convention All A values in the scan are motor-shaft degrees unless the project deliberately changes both converter and firmware conventions. grblHAL settings and calibration logic
Edge capture RP2040 must record two enter/exit pairs from the same scan direction using threshold plus hysteresis. RP2040/TMAG5273 firmware
Bounds validation The scan must reject anything other than two valid edge pairs, reject field widths outside measured limits, and reject center positions that do not repeat within tolerance. RP2040 firmware or setup calibration script
Precision rule Compute each pass center independently, then average center_1 and center_2. This improves repeatability checking and can reduce random edge noise. RP2040 firmware or setup calibration script
Final index move After computing the center, A should approach the final index from a consistent direction before setting or reporting the reference. grblHAL motion sequence

Abort / Fault Data Flow

Any abort condition in the bed-center calibration flow or A-axis magnetic homing subflow exits the current attempt. The machine must stop motion, keep or command the pen lifted, preserve diagnostic data, report the fault to the user, and require an explicit recovery decision before any retry.

Abort Source Required Handling Owner
grblHAL alarm, limit, or motion fault Stop homing motion, report alarm state to ioSender, and require manual inspection before clearing and retrying. grblHAL / ioSender
TMAG5273/RP2040 sensor or edge fault Stop the current magnetic scan, preserve the last samples and edge decisions, and leave A untrusted until the cause is corrected. RP2040 magnetic adapter
Toolhead lift fault or unknown lift state Abort before motion or stop motion immediately; the pen must remain lifted/off before retrying any homing or scan path. Toolhead MCU / grblHAL output

Step-by-Step Normal Startup Homing

Step Actor Data / Signal Requirement Failure if Missing
1. Boot safe Toolhead MCU, RP2040 adapter, RP23CNC Toolhead defaults to LIFT/OFF; grblHAL starts in known state; adapter starts sensor read loop. Reset, watchdog, or fault must not drive the pen down. Homing could drag the pen or crash the tool into the bed.
2. Connect Windows PC / ioSender USB serial or Ethernet/Telnet connection to grblHAL. ioSender can query $I, $$, $pins, and ?. User cannot start or verify homing state.
3. Lift pen ioSender, grblHAL, toolhead MCU M5 commands grblHAL to set the spindle/tool output pin to the LIFT state, then the workflow waits through a verified lift dwell. Pen/toolhead is physically clear before any X/Y/A motion. Calibration or homing motion can scrape the bed.
4. Start homing ioSender $H for full homing, or $HA for A only if single-axis homing is enabled. grblHAL homing settings include X/Y physical switches and A home input. Axes stay unreferenced or home in the wrong direction.
5. Home X/Y grblHAL / RP23CNC X/Y STEP/DIR outputs and X/Y physical home switch inputs. Switch polarity, debounce, pull-off, and travel direction are verified. False trips, no trips, or motion into hard stops.
6. Rotate A grblHAL / RP23CNC A STEP/DIR output to bed motor driver; A units are motor-shaft degrees. A direction, steps/degree, conservative seek rate, and travel limits are configured. Index is missed, bed rotates the wrong way, or motion stalls.
7. Detect magnet TMAG5273 + RP2040 adapter TMAG5273 Bx/By/Bz readings become a digital A_HOME edge. Fixed sensor height, magnet geometry, threshold, hysteresis, and output polarity are measured. Gradient noise or saturation causes inconsistent A home trips.
8. Trip A_HOME RP2040 adapter to RP23CNC Switch-like A_HOME signal into a verified RP23CNC input. Electrical interface matches the opto/input circuit and fails safe. RP23CNC input can be damaged or grblHAL never sees A home.
9. Apply offset grblHAL Homing pull-off and A home offset from measured calibration. Offset maps the trigger edge to the desired A angular index. A zero is repeatable but wrong relative to the bed.
10. Ready ioSender + grblHAL Idle/homed status. Pen remains lifted until plot G-code sends M3. Final status is checked before running converter G-code. Plot starts from an untrusted coordinate frame.

Microcontroller Requirements

RP23CNC / grblHAL Requirements

  • Firmware built with four axes so A exists.
  • X/Y physical home switches wired to verified RP23CNC inputs.
  • A home input assigned, verified with $pins, and matched to the RP2040 output polarity.
  • Homing direction, seek rate, feed rate, debounce, pull-off, and A offset recorded in settings.
  • A steps per unit configured as motor steps per degree; do not reapply the 12:1 bed ratio.
  • M5 output verified as the toolhead LIFT command before homing.

RP2040 / TMAG5273 Requirements

  • Read TMAG5273 over 3.3 V I2C/Qwiic at the fixed installed sensor height.
  • Compute field magnitude or selected vector component consistently.
  • Apply threshold and hysteresis derived from measured traces.
  • Assert a clean switch-like A_HOME output during A homing.
  • Use an output driver compatible with the selected RP23CNC input; do not connect directly until verified.
  • Expose USB serial diagnostics for setup, logging, and offset measurement.

Toolhead Controller Requirements

  • On reset, fault, watchdog expiry, or missing command, default to LIFT/OFF.
  • Interpret M5 as LIFT and M3 as ENGAGE.
  • Complete lift within a measured dwell before homing begins.
  • Later optional signal: report lifted/ready so homing can wait on proof instead of fixed dwell.

Windows Host Requirements

  • ioSender installed and able to connect to grblHAL over USB first, Ethernet later.
  • User can run M5, $H, ?, $I, $$, and $pins.
  • Separate calibration script is used for setup only: it logs magnetic traces and determines A offset.
  • Daily homing should not require the calibration script.

Setup Calibration Versus Normal Startup Homing

Mode Who Interprets Magnet Data Purpose User Action
Setup / maintenance calibration Windows calibration script using RP2040 USB diagnostic stream. Measure the gradient, determine threshold/hysteresis, find edge-to-center offset, and validate repeatability. Run a slow scripted scan and save the resulting constants.
Normal startup homing RP2040 adapter converts the magnetic profile into A_HOME; grblHAL treats it like a switch. Reference X, Y, and A without manual coordination. Connect ioSender, lift pen, run $H.

Verification Gates Before Normal Startup Homing

Do Not Skip

  • Do not connect the RP2040 A_HOME output to RP23CNC until voltage/current/polarity are verified.
  • Do not record A constants until magnet traces are repeatable at the fixed sensor height.
  • Do not run homing with the pen down.
  • Do not assume Web Builder pin mapping; verify with $pins and physical tests.

Evidence Needed

  • RP23CNC boot banner and $pins capture.
  • Unpowered STEP/DIR/ENABLE and limit-input tests.
  • TMAG5273 trace over the outer magnet with entry/exit repeatability.
  • Measured A home offset from magnetic trigger edge to desired angular index.
  • Successful homing run with pen lifted and no tool contact.
